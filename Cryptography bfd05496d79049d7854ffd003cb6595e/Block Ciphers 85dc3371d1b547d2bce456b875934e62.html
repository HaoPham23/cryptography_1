Block Ciphers

A block cipher is made up of two algorithms EEE﻿ and DDD﻿, take key kkk﻿ as a input. These algorithms take exactly n bits of plain text (PT block) as input, then output exactly n bits of cipher text (CT block) and vice versa. So it (the algorithm block) maps n bits on input to exactly n bits on output.

  • Ex: Two block ciphers
    1. 3DES: n = 64 bits, k = 168 bits
    1. AES: n = 128 bits, k = 128, 192, 256 bits

Block ciphers are built by iteration. First, the key kkk﻿ gets expanded into k1k_1k1​﻿, k2k_2k2​﻿, … , knk_nkn​﻿ called round keys. R(ki,m)R(k_i,m)R(ki​,m)﻿ called the round function. Over n rounds, the message will be encrypted to the cipher. Different block ciphers have different number of rounds (n).

Block ciphers are much slower than stream ciphers!

Incorrect use of a PRP

There are several modes in using PRP for encryption. The simplest mode is Electronic Code Book (ECB): The message is divided into blocks, and each block is encrypted separately.

So it will not semantically secure, because the same blocks PT will be encrypted to the same blocks CT. And the attacker can have some info about PT’s pattern.
